# Авторизация и регистрация

Базовый путь: `/api/auth`

JWT-токен передаётся заголовком `Authorization: Bearer <JWT-токен>`. В токене содержатся идентификатор пользователя, логин и **роль** (`USER` или `MODERATOR`).

## Заголовок `X-From-Nginx`

Проверка заголовка включена всегда. При **прямом** запросе к Spring (например на `http://localhost:8081`) нужен заголовок **`X-From-Nginx: 1`** (без кавычек, значение ровно `1`). За **nginx** (`http://localhost`) этот заголовок добавляет сам прокси (`proxy_set_header X-From-Nginx 1`).

В **Swagger UI** к каждой операции добавлен параметр заголовка `X-From-Nginx` со значением по умолчанию `1` — в «Try it out» он уходит вместе с запросом, если не удалить.

### Примеры curl (через nginx, порт `80`)

Регистрация:

```bash
curl -s -S -X POST "http://localhost/api/auth/register" \
  -H "Content-Type: application/json" \
  -d "{\"login\":\"demo_user\",\"password\":\"secret12\",\"email\":\"demo@example.com\",\"phone\":\"+79990001122\"}"
```

Вход (получить JWT из ответа):

```bash
curl -s -S -X POST "http://localhost/api/auth/login" \
  -H "Content-Type: application/json" \
  -d "{\"login\":\"demo_user\",\"password\":\"secret12\"}"
```

Запрос к защищённому API с JWT (подставьте токен из ответа login):

```bash
curl -s -S "http://localhost/api/reports" \
  -H "Authorization: Bearer YOUR_JWT_HERE"
```

Пример прямого вызова к приложению (минуя nginx):

```bash
curl -s -S "http://localhost:8081/api/regions" \
  -H "X-From-Nginx: 1"
```

### Postman

1. Вкладка **Headers**: имя **`X-From-Nginx`**, значение **`1`** (отдельной строкой; не смешивать с `Authorization`).
2. Для защищённых ручек: **Authorization → Type: Bearer Token** и вставить JWT, либо заголовок `Authorization: Bearer <jwt>` вручную.
3. Для обычных запросов используйте `http://localhost` (nginx на `:80`). Для диагностики прямого доступа используйте `http://localhost:8081` и добавляйте `X-From-Nginx: 1`.

Если с заголовком всё равно **403**, проверьте, что заголовок отправляется отдельной строкой и содержит ровно значение `1`.

## Роли

| Роль        | Описание |
|-------------|----------|
| `USER`      | Назначается при регистрации через `POST /api/auth/register`. Видит только свои обращения (отчёты) и аналитику в их контексте. |
| `MODERATOR` | Полный доступ: все отчёты, дашборд и аналитика по системе, раздел `/api/manager/**`, смена статуса отчёта. |

При первом применении миграции БД всем уже существующим учётным записям выставляется роль **MODERATOR**, чтобы поведение инсталляций не сломалось. Новых **USER** при необходимости можно повысить до модератора прямой правкой поля `users.role` в БД (в API смены роли пока нет).

Токены, выданные **до** появления claim `role`, по умолчанию считаются **MODERATOR** до истечения срока действия.

## POST /api/auth/register

Регистрация пользователя. Пароль хешируется BCrypt, при включённом блокчейне создаётся запись.

**Тело запроса:**

| Параметр      | Тип    | Обязательный | Описание                                      |
|---------------|--------|--------------|-----------------------------------------------|
| `login`       | string | да           | Логин (3–100 символов), уникальный идентификатор аккаунта |
| `password`    | string | да           | Пароль (минимум 6 символов)                   |
| `email`       | string | да           | Email (валидный формат)                        |
| `firstname`   | string?| нет          | Имя                                           |
| `lastname`    | string?| нет          | Фамилия                                       |
| `phone`       | string | да           | Телефон (уникальный)                          |
| `walletAddress`| string?| нет         | Адрес кошелька                                |

**Ответ:** 201 — успех, 400 — ошибка валидации (логин/email/телефон заняты, некорректные данные)

| Поле             | Тип     | Описание                    |
|------------------|---------|-----------------------------|
| `success`        | boolean | Результат                   |
| `message`        | string  | Сообщение                   |
| `userId`         | long?   | ID созданного пользователя  |
| `token`          | string? | JWT токен                   |
| `blockchainVerified` | boolean | Запись в блокчейне создана |
| `role`           | string? | Роль: `USER`                |

## POST /api/auth/login

Вход по логину и паролю. Возвращает JWT токен.

**Тело запроса:**

| Параметр  | Тип    | Обязательный | Описание                     |
|-----------|--------|--------------|------------------------------|
| `login`   | string | да           | login, email, vkId или phone |
| `password`| string | да           | Пароль                       |

**Ответ:** 200 — успех, 401 — неверный логин или пароль

| Поле      | Тип     | Описание                 |
|-----------|---------|--------------------------|
| `success` | boolean | Результат                |
| `message` | string  | Сообщение                |
| `userId`  | long?   | ID пользователя          |
| `token`   | string? | JWT токен (при успехе)   |
| `role`    | string? | `USER` или `MODERATOR`   |

---

## POST /api/auth/logout

Выход из системы. Тело не требуется.

**Ответ:** 200

| Поле      | Тип     | Описание  |
|-----------|---------|-----------|
| `success` | boolean | Результат |
| `message` | string  | Сообщение |

```bash
curl -s -S -X POST "http://localhost/api/auth/logout" \
  -H "Authorization: Bearer YOUR_JWT_HERE"
```

## POST /api/auth/vk-id

Привязка `vkId` к текущему авторизованному аккаунту.

**Тело запроса:**

| Параметр | Тип    | Обязательный | Описание                  |
|----------|--------|--------------|---------------------------|
| `vkId`   | string | да           | Идентификатор VK аккаунта |

**Ответ:** 200

| Поле      | Тип      | Описание                        |
|-----------|----------|---------------------------------|
| `success` | boolean  | Результат                       |
| `message` | string   | Сообщение                       |
| `userId`  | long?    | ID пользователя                 |
| `role`    | string?  | `USER` или `MODERATOR`          |

```bash
curl -s -S -X POST "http://localhost/api/auth/vk-id" \
  -H "Authorization: Bearer YOUR_JWT_HERE" \
  -H "Content-Type: application/json" \
  -d "{\"vkId\":\"123456789\"}"
```
